# Парсер учебного плана и рабочих программ дисциплин (УП + РПД)

Парсер для извлечения данных из учебного плана (УП.xlsx) и рабочих программ дисциплин (РПД.docx) с последующим объединением информации.

## Назначение

Инструмент предназначен для автоматизации обработки учебной документации вуза:
- Извлечение списка дисциплин с трудоёмкостью из учебного плана
- Парсинг компетенций, индикаторов и результатов обучения (ЗУВ) из РПД
- Объединение данных в единый структурированный формат

## Установка зависимостей

```bash
pip install openpyxl python-docx
```

## Использование

```bash
python parse_up_rpd.py --up УП.xlsx --rpd РПД.docx --out output/
```

### Параметры

| Параметр    | Обязательный | Описание                           |
|-------------|--------------|-------------------------------------|
| `--up`      | Да           | Путь к файлу учебного плана (xlsx)  |
| `--rpd`     | Нет          | Путь к файлу РПД (docx)            |
| `--out`     | Нет          | Папка для результатов (по умолчанию: текущая) |
| `--verbose` | Нет          | Подробный вывод для отладки        |

## Выходные файлы

- `disciplines_data.json` — структурированные данные в JSON
- `disciplines_report.txt` — человекочитаемый отчёт

## Структура данных

### Формат JSON

```json
[
  {
    "name": "Название дисциплины",
    "hours": 108,
    "active": true,
    "competencies": [
      {
        "code": "УК-7",
        "description": "Способен...",
        "indicators": [
          {
            "code": "УК-7.1",
            "description": "Планирует...",
            "outcomes": {
              "Знать": "...",
              "Уметь": "...",
              "Владеть": "..."
            }
          }
        ]
      }
    ],
    "summary_outcomes": {
      "Знать": "Суммарные знания",
      "Уметь": "Суммарные умения",
      "Владеть": "Суммарные навыки"
    }
  }
]
```

### Иерархия компетенций в РПД

```
Дисциплина
└─ Компетенция (ур. 1)      — код вида «УК-7», «ОПК-3»
   └─ Индикатор (ур. 2)     — код вида «УК-7.1», «ОПК-3.2»
      └─ ЗУВ (ур. 3)        — Знать / Уметь / Владеть
```

## Архитектура парсера

### Основные модули

#### 1. `parse_up()` — парсинг учебного плана

**Файл:** `parse_up_rpd.py:317`

Извлекает дисциплины из листа `ПланСвод`:
- Читает строки с маркерами `+` (активная) или `-` (неактивная)
- Фильтрует служебные строки (Блок, Практика, ГИА и т.д.)
- Возвращает список: `{name, hours, active}`

#### 2. `parse_rpd()` — парсинг РПД

**Файл:** `parse_up_rpd.py:219`

Обрабатывает документ Word с несколькими дисциплинами:
- Определяет границы дисциплин по заголовкам «Рабочая программа»
- Извлекает текст из таблиц между разделами «КОМПЕТЕНЦИИ» и «СТРУКТУРА»
- Возвращает словарь: `{имя_дисциплины: {competencies, summary_outcomes}}`

#### 3. `_DiscParser` — конечный автомат

**Файл:** `parse_up_rpd.py:117`

Класс для line-by-line парсинга компетенций:

**Состояния:**
- Поиск компетенции (код: `УК-7`, `ОПК-3`, ...)
- Поиск индикатора (код: `УК-7.1`, `ОПК-3.2`, ...)
- Накопление ЗУВ (Знать/Уметь/Владеть)

**Ключевое решение по дубликатам:**
> В РПД каждая компетенция встречается дважды:
> 1. В списке «УК-7 - Способен…» (первое вхождение)
> 2. В блоке «ПЛАНИРУЕМЫЕ РЕЗУЛЬТАТЫ» (второе — здесь индикаторы!)
>
> При втором вхождении НЕ создаётся новый объект компетенции,
> а продолжается дописывание индикаторов в существующий.

#### 4. `build_combined()` — объединение данных

**Файл:** `parse_up_rpd.py:369`

Сопоставляет дисциплины из УП и РПД:
1. Точное совпадение по нормализованному названию
2. Частичное совпадение (подстрока)
3. Fuzzy-матчинг (порог: 0.82) через `difflib.SequenceMatcher`

### Регулярные выражения

| Константа  | Паттерн                          | Назначение          |
|------------|----------------------------------|---------------------|
| `_RE_IND`  | `^[А-ЯёA-Z\-]+\d+\.\d+\s*[:–-]`  | Индикатор (УК-7.1)  |
| `_RE_COMP` | `^[А-ЯёA-Z\-]+\d+\s*[:–-]`       | Компетенция (УК-7)  |
| `_RE_ZNU`  | `^(знать\|уметь\|владеть)\s*:?\s*`| ЗУВ                 |
| `_RE_RESULT`| `В РЕЗУЛЬТАТЕ ОСВОЕНИЯ.*ДОЛЖЕН`  | Итоговый блок       |

## Вспомогательные функции

### `_clean(text)` — нормализация текста
Удаляет лишние пробелы, переносы строк.

### `_cell_texts_fast(table)` / `_cell_texts_full(table)`
Извлекают уникальный текст из таблицы с дедупликацией.

### `_znu_from_lines(lines)` — парсинг ЗУВ
Разбирает список строк на категории Знать/Уметь/Владеть.

### `_fuzzy_match(name, candidates, threshold)`
Нечёткое сопоставление названий дисциплин.

## Требования к входным файлам

### Учебный план (УП.xlsx)
- Лист с названием `ПланСвод`
- Колонки: маркер (A), название (B), часы (K)
- Строки начинаются с 4-й

### Рабочие программы (РПД.docx)
- Заголовки дисциплин: «Рабочая программа дисциплины»
- Таблицы с компетенциями после заголовка «КОМПЕТЕНЦИИ ОБУЧАЮЩЕГОСЯ»
- Структура компетенций: код — описание — индикаторы — ЗУВ
